# FOS-Agri V3 — Project Rules

## Homepage lock

The client-approved homepage is `index.html`. It must remain visually unchanged.

Do not change:

- Hero section
- Video
- Image slider
- Agenda section
- Maps section
- Animations
- Homepage layout
- Homepage content
- Homepage visual design

Allowed homepage changes:

- Safe header/menu link fixes only
- Safe submenu connection fixes only

## Phase 1 technical rules

- Keep the project static HTML/CSS/JavaScript.
- Keep GitHub Pages compatibility.
- Do not add a database.
- Do not add a backend.
- Do not add Hostinger backend dependency.
- Do not add React, Next.js, Laravel, PHP, WordPress, or package dependencies yet.

## Commit rules

- Make small, focused commits.
- Do not make large rewrites.
- Do not remove existing working assets.
- Summarize exactly which files changed.

## Future migration rule

If the project later moves to Next.js, migrate the approved homepage carefully without redesigning it.
